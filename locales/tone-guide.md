# ALGO — Guide de voix (Tone Guide)

## Source machine (prompts & cohérence)

Les règles structurées pour l’IA et la revue de copy vivent dans le code : **`src/lib/copy/algo-voice.ts`** (ADN, checklist, couche système modèle). Ce fichier est la référence quand le guide humain et le comportement ALGO AI doivent rester alignés.

## Principe fondamental

ALGO parle comme un radar, pas comme un assistant.
Court. Direct. Précis. Jamais condescendant.
Expert **calme** : maîtrise sans arrogance; chaque mot utile (« l’appli silencieuse qui parle à tout le monde » = peu de bruit, beaucoup de sens).

## Ce qu'ALGO dit

| ❌ Interdit                    | ✅ ALGO                                      |
|-------------------------------|----------------------------------------------|
| "Aucun résultat trouvé."      | "Le radar est silencieux pour l'instant."    |
| "Oops ! Quelque chose a raté."| "Signal perdu. On réessaie."                |
| "Bienvenue sur ALGO !"        | "Vois ce qui arrive… avant tout le monde."  |
| "Chargement en cours…"        | "Lecture des signaux…"                       |
| "Vous n'avez pas de favoris." | "Ton radar est vide. Explore et sauvegarde."|
| "Erreur réseau."              | "Signal perdu. On réessaie."                |

## Règles

1. **Court** — une phrase, pas un paragraphe.
2. **Actif** — pas de voix passive.
3. **Jamais "Oops!"** — interdit.
4. **Jamais condescendant** — surtout dans le Fail Lab.
5. **Tutoiement** — toujours, en français.
6. **Métaphore radar** — signaux, fréquences, ondes, réception.

## Fail Lab — Règle spéciale

Le ton du Fail Lab est analytique et neutre.
On analyse l'échec, on n'humilie pas.
On dit "Pourquoi ça n'a pas décollé", jamais "Pourquoi c'était nul".
